查看文件的两个版本之间的差异如何帮助你了解引入的错误？

如果以前的文件是没有错误的，那错误就在更改过后的地方。所以对比差异就可以满足找到错误的地方。

从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？

能访问整个历史记录，就可以观察到文件每一步的变化过程，从中了解文件的进程，所以能提高效率。

 你认为，手动选择何时创建提交（像git那样）与自动保存版本（如Google docs那样）各有何优缺点？

 手动提交更灵活，并且具有人的思想，更有逻辑性。但是那样就要求使用人员有一个好的习惯，而且即使有这个好的习惯也容易忘记。
 自动保存会创建更多的历史记录，而且没有逻辑性，但更简单更直接。

 你认为一些版本控制系统（例如Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？

 git是针对程序的，各个文件的连接性更大。而Google Docs针对的文件不同。

 如何使用命令 git log 和 git diff 来查看文件的历史记录？

 git log 会显示每次的commit ，按下键查找。 git diff  对比两个commit id的不同之处。

 使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？

 以前的版本都会被保存下来，即使出现错误也可以比较轻松的更改。
